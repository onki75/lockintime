# ゲーミフィケーション

## 設計思想

Duolingoのゲーミフィケーション手法を参考にしつつ、サイトブロッカー特有の課題に適応させる。

### Duolingoとの構造的な違い

| | Duolingo | LockInTime |
|---|---|---|
| 行動タイプ | **能動的**（レッスンをやる） | **受動的**（誘惑に耐える） |
| セッション | 3-5分の集中 | 終日の行動 |
| 性質 | 社会的・競争的 | 個人的・プライベート |
| 報酬設計 | 行動するほど報酬 | **何もしない**ことが報酬 |

**核心課題**: 「何もしないことにポジティブなフィードバックを与える」

### 不採用のDuolingo要素

| 要素 | 不採用の理由 |
|------|------------|
| リーグ/リーダーボード | ブロック対象サイトは**プライバシー情報**。他人と比較する動機も薄い |
| XP（経験値） | 「何もしない」ことにXPを付与するのは不自然。レスキューパスに統一 |
| ハート制（ライフ） | ブロッカーで「ミスしたら使えない」は本末転倒 |
| フレンドクエスト | Chrome拡張でソーシャル機能はオーバースコープ |

## ストリーク強化

詳細なロジック・UI・タスクは [streak.md](./streak.md) を参照。ここではゲーミフィケーション観点の拡張点を記載。

### 表示場所の拡大

現行: ブロックページのみ → **常時可視化**に変更

| 表示場所 | 内容 |
|----------|------|
| **ポップアップ最上部** | 全体ストリーク日数 + 炎アイコン |
| **ブロックページ** | サイト別 + 全体（現行通り） |
| **拡張アイコンバッジ** | P2以降はストリーク日数を表示（有効ルール数から切替） |

**心理原理**: ストリークの常時可視化が損失回避とサンクコスト効果を強化。Duolingoではストリークがアクティブなユーザーのリテンション率が3倍。

### マイルストーン

ストリーク継続に対する段階的な報酬を設ける。

| 日数 | イベント | 報酬 |
|------|---------|------|
| 7日 | セレブレーション演出 | レスキューパス 1枚 |
| 30日 | セレブレーション演出（大） | レスキューパス 2枚 |
| 100日 | セレブレーション演出（特大） | レスキューパス 3枚 + ストリークフリーズ上限拡張（最大3個） |
| 365日 | セレブレーション演出（特別） | レスキューパス 5枚 + ストリークフリーズ上限拡張（最大5個） |

**心理原理**: 段階的コミットメント（foot-in-the-door技法）。小さなマイルストーンから始まり、次第に大きな報酬が得られるエスカレーション効果。

### ストリーク修復（Earn Back）

ストリークを失った翌日に「追加の努力」で前日を修復できる救済措置。

- ストリークを失った**翌日のみ**有効（2日以上の連続失敗は修復不可）
- 条件: その日、全ルールのブロックを守りきる + **追加で3時間ブロック延長を維持**
  - 例: 普段18時にブロック解除 → 21時まで維持して初めて修復
- 達成すると前日が「修復」（黄色）として記録、ストリーク継続
- ポップアップに「ストリーク修復チャンス!」バナーを表示

**心理原理**: 完全な喪失ではなく「取り戻せる」という希望が離脱を防止。Duolingoの Streak Earn Back と同じ原理。

## レスキューパス

ストリーク継続や目標達成で獲得するアプリ内通貨。複数の用途に使える。

### 獲得条件

| 条件 | パス数 | 頻度 |
|------|--------|------|
| 7日連続ストリーク | 1枚 | 週1回 |
| マイルストーン達成（30日） | 2枚 | 低頻度 |
| マイルストーン達成（100日） | 3枚 | 低頻度 |
| マイルストーン達成（365日） | 5枚 | 低頻度 |
| デイリーチャレンジ（ゴールド） | 1枚 | 日次（P2+） |

### 使い道（3択）

| 用途 | 消費数 | 効果 |
|------|--------|------|
| **一時解除** | 1枚 | ブロックを一時解除。ストリーク記録は「一時解除」（黄色）、失敗にはならない |
| **ストリークフリーズ装備** | 1枚 | 翌日サボってもストリーク維持。最大2個ストック（100日達成で3個、365日で5個） |
| **マスコットの餌** | 1枚 | マスコットが成長する（P3で実装） |

**心理原理**: Duolingoのジェムと同様の「単一通貨 + 複数用途」設計。「使うか貯めるか」のジレンマが自制を強化。エンダウメント効果（自分で稼いだものへの愛着）も作用。

### ストリーク記録の3状態

| 状態 | 色 | 条件 |
|------|-----|------|
| 成功 | 緑 | 全ルールを守りきった |
| 一時解除 | 黄色 | パス消費して解除 / ストリーク修復 |
| 失敗 | 赤 | パスなしで制限を破った |

### データモデル

```typescript
interface RescuePass {
  available: number       // 現在の保有パス数
  frozenCount: number     // 装備中のストリークフリーズ数
  frozenMax: number       // ストリークフリーズ上限（デフォルト2、マイルストーンで拡張）
  totalEarned: number     // 累計獲得数
  totalUsedBypass: number // 累計使用数（一時解除）
  totalUsedFreeze: number // 累計使用数（フリーズ）
  totalUsedFeed: number   // 累計使用数（餌）
}
```

## デイリーチャレンジ

毎日3つのチャレンジを提示し、「受動的な行動」にアクティブな目標を与える。

### チャレンジ一覧

| 難易度 | チャレンジ例 | 報酬 |
|--------|-------------|------|
| ブロンズ（易） | 今日ブロックを1回も解除しない | マスコット笑顔演出 |
| シルバー（中） | ブロック対象サイトへのアクセス試行を3時間以上ゼロに | マスコット演出 + 小セレブレーション |
| ゴールド（難） | 全ルールを24時間守りきる | レスキューパス 1枚 + セレブレーション |

### 設計ポイント

- 毎日0:00にリセット、3つ提示（ブロンズ / シルバー / ゴールド）
- **段階的難易度**: ブロンズは達成しやすく、ゴールドは1日かかる
- ポップアップに進捗表示（「あと1時間で達成!」）
- チャレンジの内容はユーザーの制限タイプに応じて動的生成
  - 例: 使用時間制限を設定中 → 「今日の使用時間を上限の50%以内に抑える」
  - 例: 完全ブロックのみ → 「今日ブロックを1回も解除しない」

**心理原理**: Duolingoのデイリークエスト（DAU 25%増加）と同じ原理。「何もしない」を能動的な目標に変換するのが核心。段階的報酬 + チェスト開封的な報酬体験。

### プラン別

| | Free | Pro / Cloud |
|---|---|---|
| チャレンジ数/日 | 1つ（ブロンズのみ） | 3つ（ブロンズ / シルバー / ゴールド） |
| パス獲得 | ❌ | ゴールド達成で1枚 |

### データモデル

```typescript
interface DailyChallenge {
  id: string
  date: string            // "2026-03-18"
  tier: 'bronze' | 'silver' | 'gold'
  type: string            // チャレンジタイプ
  target: number          // 目標値
  current: number         // 現在値
  completed: boolean
  completedAt: number | null
}

interface DailyChallengeState {
  challenges: DailyChallenge[]
  lastGeneratedDate: string
}
```

## セレブレーション演出

日常的な小さい報酬演出から、大きなマイルストーン演出まで。

### 演出一覧

| タイミング | 演出 | 規模 |
|-----------|------|------|
| ブロックページ表示時 | カウンター増加アニメ「今日N回ブロック成功!」 | 小 |
| デイリーチャレンジ達成 | チェスト開封アニメ + 報酬表示 | 中 |
| ストリーク7日到達 | レスキューパス獲得演出 + 紙吹雪 | 中 |
| マスコットレベルアップ（P3） | 進化アニメーション（3-5秒） | 大 |
| ストリークマイルストーン（30/100/365日） | フルスクリーン紙吹雪 + 共有ボタン | 特大 |

**心理原理**: Duolingoの「juicy feedback」。即座の正のフィードバックが行動-報酬の神経回路を強化。日常的な小さい演出が習慣ループを維持し、大きな演出がマイルストーンの達成感を増幅する。

## マスコット育成

マスコットの感情がリアルタイムでユーザーの行動を反映する。たまごっち的な育成要素。

### 成長テーブル

| レベル | 必要な餌 | 見た目 | 累計日数目安 |
|--------|---------|--------|------------|
| 🥚 たまご | 0 | 初期状態 | 0日 |
| 🐣 ひよこ | 3個 | 小さい盾キャラ | 21日 |
| 🛡 見習い | 7個 | 通常の盾キャラ | 49日 |
| ⚔️ 戦士 | 15個 | 装飾付き盾 | 105日 |
| 👑 マスター | 30個 | 王冠付き盾 | 210日 |

- パスを一時解除に使うとマスコットが悲しむ（退化はしない）
- マスコットはポップアップのカレンダー上部に常駐表示
- レベルアップ時にセレブレーション演出

### 感情デザイン

Duoフクロウの表情変化を参考に、リアルタイムの状態反映。

| トリガー | マスコットの反応 | 表示場所 |
|----------|----------------|----------|
| ストリーク継続中 | 笑顔、応援メッセージ | ポップアップ |
| ブロックページ表示時 | 誇らしい顔「守ってるね!」 | ブロックページ |
| 一時解除しようとした時 | 心配顔「本当に必要?」 | 一時解除ダイアログ |
| 一時解除実行（パスあり） | 少し悲しい顔 | 一時解除後 |
| 一時解除実行（パスなし） | かなり悲しい顔 | 一時解除後 |
| デイリーチャレンジ達成 | お祝い + 紙吹雪 | ポップアップ |
| レベルアップ | 特別演出（進化アニメ） | ポップアップ |
| ストリーク危機（23時台で未達成） | 焦り顔 + 通知 | ブラウザ通知 |
| 長期離脱（3日以上ストリーク切れ） | 涙目「戻ってきて...」 | ポップアップ |

**心理原理**: 擬人化効果 + パラソーシャル関係。「マスコットを悲しませたくない」という感情が自制力を補強。Duolingoでは「You made Duo sad」系のメールが通常より5-8%高いレッスン完了率を達成。

### データモデル

```typescript
interface MascotState {
  level: number           // 0-4
  feedCount: number       // 累計餌の数
  lastFedAt: number | null
}
```

## UIモード切替

ビジネス向けにマスコットなしの「シンプルモード」を用意する。

| | シンプルモード | マスコットモード |
|---|---|---|
| ポップアップ | ストリーク数値 + カレンダー（色のみ） | マスコット表示 + 育成UI + カレンダー（スタンプ） |
| ブロックページ | 名言のみ | マスコットの表情 + 名言 |
| 一時解除 | 反省カード + 長押し確定 | 同じ + マスコットの反応 |
| レスキューパス | 一時解除 or フリーズのみ | 同左 + 餌として使用 |
| デイリーチャレンジ | テキスト表示のみ | マスコット演出付き |
| 設定 | 表示設定タブで切替 | デフォルト |

- 設定ページの「表示設定」タブで切替: ● マスコットモード / ○ シンプルモード
- シンプルモードでもストリーク・パス・チャレンジは機能する（マスコット演出がないだけ）
- マスコットモードがデフォルト

## フェーズ計画

マスコットは開発コストが高い（イラスト・アニメーション）ため、先にストリーク + チャレンジ + パスで**仕組みの骨格**を作り、マスコットは後から**演出レイヤー**として被せる。マスコットなしでもゲーミフィケーションが機能する設計。

| フェーズ | 要素 | 内容 |
|---------|------|------|
| **P2** | ストリーク強化 | マイルストーン、ストリーク修復（Earn Back） |
| **P2** | レスキューパス | 一時解除 + ストリークフリーズの2用途 |
| **P2+** | デイリーチャレンジ | 3段階チャレンジ、動的生成 |
| **P2+** | セレブレーション演出 | ブロック成功カウンター、チェスト開封、紙吹雪 |
| **P3** | マスコット育成 | 成長・進化・感情デザイン |
| **P3** | UIモード切替 | マスコットモード / シンプルモード |
| **Future** | マスコットカスタマイズ | 色、アクセサリー等 |
| **Future** | SNS共有 | マイルストーン達成時の共有ボタン |

| 機能 | Free | Pro | Cloud |
|------|------|-----|-------|
| ストリーク（数値のみ） | ✅ | ✅ | ✅ |

## 差別化ポイント

- 競合のサイトブロッカーにはマスコット育成要素がない
- 「ブロックを続ける = マスコットが育つ」というポジティブな動機付け
- レスキューパスの「使うか貯めるか育てるか」の3択ジレンマが自制を強化
- デイリーチャレンジが「何もしない」を「能動的な目標」に変換
- SNSでの共有ネタにもなる（「マスコットが戦士に進化した!」）
