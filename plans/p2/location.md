# 位置情報制限

特定の場所にいる間だけサイトをブロックする。
「職場にいる間はSNSブロック」のようなユースケースに対応。

---

## 機能

- 名前付きの場所を登録（名前・緯度・経度・半径）
- ブロックルールに場所を紐付け
- 圏内にいる間 → ブロック有効、圏外 → 無効
- 場所の登録方法: 現在地から取得 or 手動入力

---

## Chrome API

- Geolocation API — 位置情報取得（Web標準、Chrome権限不要）
- `chrome.alarms` — 定期的な位置チェック（5分間隔）
- `declarativeNetRequest` — 圏内/圏外でルール有効化/無効化

---

## タスク

### Background
- `src/background/location-checker.ts` 新規作成
  - `chrome.alarms` で5分間隔で位置情報を取得
  - 登録場所の圏内/圏外を判定（緯度・経度・半径による距離計算）
  - 圏内 → ルール有効化 / 圏外 → 無効化
- `rule-sync.ts` を拡張: `location` タイプ対応

### Geolocation
- Background SWからは直接使えないため、offscreen documentまたはオプションページから取得
- 取得した位置情報をstorageに保存し、Background SWが読み取る

### 場所の管理
- `storage.ts` に場所のCRUD関数を追加
  - `addLocation(name, lat, lng, radius)`
  - `removeLocation(id)`
  - `updateLocation(id, partial)`

### UI
- 設定ページ「場所の管理」セクション:
  - 場所一覧（名前・座標・半径・削除ボタン）
  - 「現在地から追加」→ Geolocation API → 名前と半径を入力して保存
  - 「手動で追加」→ 緯度・経度・名前・半径を入力
- サイト追加ダイアログ: 位置情報制限の選択肢（登録場所のドロップダウン）
- ポップアップ: 「職場で有効」のようなステータス表示
- ブロックページ: 「この場所ではアクセスできません」メッセージ

## 完了条件

- 場所を登録し、圏内でのみブロックが有効になる
- 圏外に出るとブロックが自動解除される
- 位置情報の許可ダイアログが適切に表示される
- `npm run build` / `npm test` が通る
