# Phase 3: 高度な機能

位置情報制限とストリーク機能で体験を強化する。

---

## ゴール

- 場所を登録し、その場所にいる間だけサイトをブロックできる
- ストリーク（サイト別 + 全体）がブロックページに表示される
- ヒートマップ（GitHub草風）or 数値の切替ができる

---

## タスク

### 3-1. 位置情報制限

**Background:**
- `src/background/location-checker.ts` 新規作成
- chrome.alarms で定期的（5分間隔）に位置情報を取得
- 登録された場所の圏内/圏外を判定（緯度・経度・半径による距離計算）
- 圏内 → 対象ルールのdeclarativeNetRequestルールを有効化
- 圏外 → 無効化
- `rule-sync.ts` を拡張: `location` タイプ対応

**場所の管理:**
- `storage.ts` に場所のCRUD関数を追加
  - `addLocation(name, lat, lng, radius)`
  - `removeLocation(id)`
  - `updateLocation(id, partial)`

**Geolocation API:**
- Background SWからは直接使えないため、offscreen documentまたはポップアップ/オプションページから取得
- 取得した位置情報をstorageに保存し、Background SWが読み取る

**UI:**
- 設定ページ「場所の管理」セクション:
  - 場所一覧（名前・座標・半径・削除ボタン）
  - 「現在地から追加」ボタン → Geolocation API → 名前と半径を入力して保存
  - 「手動で追加」ボタン → 緯度・経度・名前・半径を入力
- サイト追加ダイアログ: 位置情報制限の選択肢を有効化（登録場所のドロップダウン）
- ポップアップ: 「職場で有効」のようなステータス表示
- ブロックページ: 「この場所ではアクセスできません」メッセージ

### 3-2. ストリーク機能

**ロジック:**
- `src/lib/streak.ts` 新規作成
- ストリーク判定:
  - サイト別: その日、対象サイトの制限を破らなかった → success
  - 全体: 全ルールでsuccessだった → global success
- 連続日数の計算: StreakRecord[]を末尾から走査し、連続successをカウント
- 最長記録の計算: 全期間での最長連続successを計算

**日次更新:**
- DailyStatsの日次リセット時（Phase 2で実装済み）にストリーク判定を実行
- 前日のDailyStatsを元に StreakRecord を生成
- StreakDataに追記

**ヒートマップコンポーネント:**
- `src/components/Heatmap.tsx` 新規作成
- GitHub草風のグリッド表示（12週分 = 84日）
- 日ごとに success → 緑、failure → 灰色
- 連続日数が長いほど緑が濃くなる（4段階）

**UI:**
- ブロックページにストリーク表示:
  - サイト別ストリーク（そのサイトの連続日数 + ヒートマップ or 数値）
  - 全体ストリーク（全ルール総合の連続日数 + ヒートマップ or 数値）
  - ヒートマップ/数値の切替ボタン
- 設定ページ:
  - 「表示設定」セクションでstreakDisplayMode切替（ヒートマップ / 数値）

## 完了条件

- 場所を登録し、圏内でのみブロックが有効になる
- 圏外に出るとブロックが自動解除される
- ストリークがブロックページに正しく表示される（サイト別 + 全体）
- ヒートマップ表示と数値表示を切り替えられる
- ヒートマップが12週分の日別データを草風に表示する
- `npm run build` が通る
- `npm test` が通る
