# 権限の正当性説明（Chrome ウェブストア審査用）

Developer Dashboard の「プライバシー慣行 (Privacy practices)」タブで、各権限ごとに用途を入力する。以下はそのまま貼り付けられる説明文（日本語）。英語版が必要な場合は併記推奨。

> ⚠️ `<all_urls>` のホスト権限・`geolocation`・`declarativeNetRequest` は審査が慎重になりやすい。下記の説明で「ユーザーが指定したブロック対象サイトを評価するため」という目的を明確にしている。

## 単一目的（Single purpose）の説明

LockInTime は、ユーザーが指定したウェブサイトをブロック・制限し、PC のスクリーンタイムを可視化することで集中を支援する拡張機能です。すべての機能はこの単一目的（集中のためのサイト制限とスクリーンタイム管理）に紐づいています。

## 各権限の説明

### `storage`
ユーザーのブロック設定、ルール、スクリーンタイムの記録、連続達成日数（ストリーク）を端末内（chrome.storage.local）に保存するために使用します。データは端末から外部に送信されません。

### `alarms`
1日ごとのカウント・利用時間のリセット、クールダウンや時間帯制限のスケジュール判定など、時間に基づくルールを定期的に評価するために使用します。

### `declarativeNetRequest`
ユーザーが指定したサイトへのアクセスをブロック／リダイレクトするために使用します。Chrome の宣言的ルールで実装しており、拡張機能がページ内容を読み取ることはありません。

### `tabs`
現在開いているタブの URL を確認し、制限対象サイトかどうかを判定して、必要に応じてブロックページへ誘導するために使用します。

### `webNavigation`
ページ遷移のタイミングを検知し、制限対象サイトへの移動時に遅延ゲートやブロックを適用するために使用します。

### `geolocation`
「特定の場所にいるときだけ／いないときだけブロックする」位置情報ベースの制限ルール（location タイプ）を評価するためにのみ使用します。取得した座標は端末内でのルール判定に使うだけで、外部サーバーには送信しません。

### `offscreen`
Service Worker から直接実行できない処理（位置情報の取得など）を、オフスクリーンドキュメント上で実行するために使用します。

### `favicon`
ブロック対象サイトの一覧で、各サイトのファビコン（アイコン）を表示するために使用します。

### ホスト権限 `<all_urls>`
ユーザーは任意のウェブサイトをブロック対象として登録できます。どのサイトが対象になるか事前に分からないため、全 URL に対してブロックルールを適用できる必要があります。拡張機能がページの内容を収集・送信することはありません。

## リモートコード（Remote code）
リモートコードは使用していません。すべてのコードは拡張機能パッケージに同梱されています。
