# 今日ここまで耐えた時間

## 概要

ポップアップに「今日ここまで耐えた時間」を表示する。
「今日失った時間」（ネガティブ指標）と対になるポジティブ指標。

---

## 目的

- ユーザーのモチベーション維持
- 「失った時間を減らして、耐えた時間を増やす」というインセンティブ
- 小さな成功体験の可視化（「今日3時間耐えた」）

---

## 表示イメージ

```
┌──────────────────────────────────────┐
│ 今日失った時間  ⓘ                     │
│ ┌──────────┐  youtube.com    45分     │
│ │  1時間23分│  x.com         23分     │
│ │   ↑ +15分 │  tiktok.com    15分     │
│ └──────────┘                          │
│                                      │
│ 🛡 今日ここまで耐えた時間: 5時間37分    │
└──────────────────────────────────────┘
```

---

## 計算方法の候補

### 案A: 起床時刻ベース
```
耐えた時間 = (現在時刻 - 起床時刻) - ブロック対象サイトの合計利用時間
```
- 起床時刻の設定が必要（デフォルト7:00?）
- 正確だが設定コストがある

### 案B: 0時ベース（シンプル）
```
耐えた時間 = (現在時刻 - 00:00) - ブロック対象サイトの合計利用時間
```
- 設定不要
- 寝ている時間も含まれるので数字が大きくなる

### 案C: ブラウザ利用時間ベース
```
耐えた時間 = ブラウザ利用時間 - ブロック対象サイトの合計利用時間
```
- 最も正確（「ブラウザを使っている間、誘惑に耐えた」）
- tab-trackerで全タブの利用時間を計測する必要がある（追加実装）

### 案D: ブロック有効時間ベース
```
耐えた時間 = ブロックルールが有効だった時間 - アクセスした時間
```
- ルールのON/OFFタイミングを記録する必要がある

### 推奨: 案B（シンプル）
- 実装コスト最小
- DailyStats.durationsの合計を引くだけ
- ユーザーにとって直感的（「今日の24時間のうち、X時間は耐えた」）

---

## データソース

- `DailyStats.durations` — ブロック対象サイトの累積秒数（既に計測済み）
- `Date.now()` — 現在時刻
- 追加のデータ取得は不要

---

## 実装メモ

- ポップアップの`ScreenTimeSection`内 or 直下に配置
- `todayMinutes`（失った時間）は既に計算済み
- `耐えた分 = (今日の経過分) - todayMinutes`
- 0未満にならないよう `Math.max(0, ...)` でクランプ
