# ロックモード・チャレンジ・Nuclear Option

衝動的なブロック解除を防ぐための3段階の保護機能。
ユーザーが自分の意志力に合わせて保護レベルを選べる。

---

## 3つの保護レベル

| レベル | 名前 | 解除方法 | 強度 |
|--------|------|----------|------|
| 1 | **パスワード** | パスワード入力で即解除 | 低 |
| 2 | **テキストチャレンジ** | 指定文言の手入力 + 解除理由の記入 | 中 |
| 3 | **Nuclear Option** | 設定時間まで絶対に解除不可 | 最高 |

ユーザーが設定時にどのレベルを使うか選択する。
どのレベルでも**24時間遅延アンロック**がパスワード忘れ時の回復手段として利用可能（Nuclear Optionを除く）。

---

## レベル1: パスワード

- パスワードを設定すると、以下の操作にパスワード入力が必要:
  - ルールの無効化・削除
  - 一時解除の利用
  - ロックモード自体の解除
- パスワードはSHA-256でハッシュ化して `chrome.storage.local` に保存
- パスワード忘れ → 24時間遅延アンロックで回復

## レベル2: テキストチャレンジ

- 解除時に**指定文言の正確な手入力**が必要
- さらに**解除理由の記入**が必須
- 記入内容はストリークの記録に保存される（後で振り返り可能）
- タイピングの手間 + 理由を考える時間が衝動を冷ます
- チャレンジ失敗 → 24時間遅延アンロックで回復

## レベル3: Nuclear Option

- 設定時に**ロック期間**を指定（1時間 / 6時間 / 12時間 / 24時間 / カスタム）
- ロック期間中は**いかなる方法でも解除不可**
- 24時間遅延アンロックも無効
- パスワードもチャレンジも受け付けない
- 拡張の再インストールでも解除不可（Cloudプランの場合、Firestore側でロック状態を管理）
- 期間終了後に自動解除

---

## 制限事項

Chrome拡張の技術的制約として、以下は防げない:
- ブラウザの拡張管理画面（chrome://extensions）からの無効化・削除
- シークレットモードでのアクセス
- 別ブラウザの使用

これらは完全なペアレンタルコントロールではなく、**自制支援**の位置づけ。

※ Nuclear Optionでも拡張削除は防げない。ただし再インストール時にCloudプランならFirestoreからロック状態を復元する。

### パスワード回復（24時間遅延アンロック）

パスワードまたはチャレンジを通せない場合の回復手段。
**Nuclear Option中は使用不可。**

```
┌──────────────────────────────────────┐
│ ロック解除リクエスト                   │
├──────────────────────────────────────┤
│ パスワードなしでロックを解除するには   │
│ 24時間待つ必要があります。            │
│                                      │
│ リクエスト後、24時間経過すると        │
│ ロックが自動解除されます。            │
│ （いつでもキャンセル可能）            │
│                                      │
│       [キャンセル] [リクエスト]       │
└──────────────────────────────────────┘
```

## 完了条件

- 3つの保護レベル（パスワード/チャレンジ/Nuclear）が設定画面から選べる
- パスワード: 入力で即解除、ハッシュ化保存
- チャレンジ: 文言入力 + 理由記入で解除、記録がストリークに保存
- Nuclear: 設定期間中は一切解除不可
- パスワード/チャレンジの忘れ → 24時間遅延アンロック（Nuclear以外）
- `npm run build` / `npm test` が通る
