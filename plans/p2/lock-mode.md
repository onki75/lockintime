# ロックモード（パスワード保護）

設定変更やブロック解除にパスワードを要求する。
ユーザーが衝動的にブロックを解除してしまうのを防ぐ。

---

## 機能

- パスワードを設定すると、以下の操作にパスワード入力が必要になる:
  - ルールの無効化・削除
  - 一時解除の利用
  - ロックモード自体の解除
- パスワード未設定時は従来通り自由に操作可能
- 拡張の無効化/削除はブラウザ側の操作なので防げないが、「拡張が無効化されました」の通知は可能

---

## 制限事項

Chrome拡張の技術的制約として、以下は防げない:
- ブラウザの拡張管理画面（chrome://extensions）からの無効化・削除
- シークレットモードでのアクセス
- 別ブラウザの使用

これらは完全なペアレンタルコントロールではなく、**自制支援**の位置づけ。

---

## タスク

### パスワード管理
- パスワードはハッシュ化して `chrome.storage.local` に保存
- SHA-256等でハッシュ化（平文保存しない）

### UI
- 設定ページに「ロックモード」セクションを追加
  - パスワード設定/変更/解除
- パスワード入力ダイアログ（保護された操作の前に表示）

## 完了条件

- パスワードを設定するとルール変更にパスワードが必要になる
- パスワードなしでは保護された操作ができない
- パスワードがハッシュ化されて保存される
- `npm run build` / `npm test` が通る
