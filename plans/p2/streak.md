# ストリーク

制限を守った連続日数を記録・表示する。モチベーション維持のための機能。
サイト別 + 全体の両方を追跡し、複数の場所に表示する。

ゲーミフィケーション全体の設計は [gamification.md](./gamification.md) を参照。

---

## 機能

- **サイト別ストリーク**: そのサイトの制限を守った連続日数
- **全体ストリーク**: 全ルールの制限を守った連続日数
- **表示形式**: GitHub草風ヒートマップ or 数値（設定で切替）
- **表示場所**: ポップアップ最上部 + ブロックページ + 拡張アイコンバッジ
- **最長記録**: 全期間での最長連続日数を記録
- **マイルストーン**: 7日 / 30日 / 100日 / 365日でセレブレーション演出 + レスキューパス獲得
- **ストリーク修復（Earn Back）**: 失った翌日に追加の努力で前日を修復可能
- **ストリークフリーズ**: レスキューパスで装備、翌日サボってもストリーク維持

---

## 成功の定義

「成功 = 制限内に収まった」。制限タイプ別:

| タイプ | 成功 | 失敗 |
|--------|------|------|
| 完全ブロック | アクセスしなかった | ブロックページが表示された |
| 使用時刻制限 | 制限時間帯にアクセスしなかった | 制限時間帯にブロックされた |
| 使用回数制限 | 上限内でアクセス | 上限を超えてブロックされた |
| 使用時間制限 | 上限内で閲覧 | 上限を超えてブロックされた |
| クールダウン | クールダウン中にアクセスしなかった | クールダウン中にブロックされた |
| 遅延アクセス | 常に成功（ブロックではないため） | - |
| 位置情報制限 | 制限場所でアクセスしなかった | 制限場所でブロックされた |

※ 遅延アクセスは「待機後にサイトを閲覧」しても成功扱い。ストリーク対象だが失敗にはならない。

### Cloud同期時のマージルール
- 同じ日にデバイスAで成功、デバイスBで失敗 → **失敗が優先（AND）**
- どちらかのデバイスで制限を破ったら、その日は失敗

## タスク

### ロジック
- `src/lib/streak.ts` 新規作成
  - ストリーク判定:
    - サイト別: その日、対象サイトの制限を破らなかった → success
    - 全体: 全ルールでsuccessだった → global success
    - パス消費して一時解除した日 → bypass（ストリーク継続）
    - パスなしで制限を破った → failure
  - 連続日数の計算: StreakRecord[]を末尾から走査（success + bypass を連続とカウント）
  - 最長記録の計算: 全期間での最長連続（success + bypass）
  - ストリークフリーズ判定: フリーズ装備中にfailureの日 → bypass に変換、フリーズ1個消費
  - ストリーク修復判定: 前日がfailure + 当日の追加条件を達成 → 前日をbypassに更新
  - マイルストーン判定: 連続日数が閾値に到達 → レスキューパス付与 + セレブレーション

### 日次更新
- DailyStatsの日次リセット時にストリーク判定を実行
- 前日のDailyStatsを元に StreakRecord を生成
- StreakDataに追記
- ストリークフリーズの自動消費チェック
- マイルストーン到達チェック + レスキューパス付与

### ヒートマップコンポーネント
- `src/components/Heatmap.tsx` 新規作成
  - GitHub草風のグリッド表示（12週分 = 84日）
  - 日ごとに success → 緑、bypass → 黄色、failure → 灰色
  - 連続日数が長いほど緑が濃くなる（4段階）
  - **アクセシビリティ**: 色だけでなくパターン（斜線 = 失敗、ドット = 一時解除）+ ツールチップ（「3月16日: 成功」）で情報を伝える（WCAG 2.1 AA準拠）

### UI
- **ポップアップ最上部**にストリーク表示:
  - 全体ストリーク日数 + 炎アイコン
  - ストリーク修復チャンス表示（前日失敗時）
- **ブロックページ**にストリーク表示:
  - サイト別（連続日数 + ヒートマップ or 数値）
  - 全体（連続日数 + ヒートマップ or 数値）
  - 表示形式の切替ボタン
- **拡張アイコンバッジ**: ストリーク日数を表示（有効ルール数から切替）
- 設定ページ:
  - 「表示設定」で streakDisplayMode 切替

## 完了条件

- ストリークがポップアップ + ブロックページに正しく表示される（サイト別 + 全体）
- ヒートマップ表示と数値表示を切り替えられる
- ヒートマップが12週分の日別データを草風に表示する（3色: 緑/黄/灰）
- 日次更新で前日の判定が正しく行われる（success / bypass / failure の3状態）
- ストリークフリーズが正しく消費される
- ストリーク修復（Earn Back）が翌日のみ有効に機能する
- マイルストーン到達時にレスキューパスが付与される
- 拡張アイコンバッジにストリーク日数が表示される
- `npm run build` / `npm test` が通る
