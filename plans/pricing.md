## プラン詳細

### Free（無料）
ログイン不要で使える基本プラン。

- 完全ブロック + 使用時刻制限（曜日別スケジュール含む）
- **単体ルール: 5サイトまで**
- プリセットからは個別サイトを選んで単体ルールとして追加（5件上限内）
- グループ機能なし
- ブロックページ表示

**目的**: まず使ってもらい、価値を体験してもらう。

**初回トライアル**: インストールから**7日間はPro全機能を無料体験**可能（グループ含む）。カード登録不要。期間終了後に自動でFreeに移行（ダウングレードUIを表示）。トライアル管理の詳細は後述の「トライアル管理」セクションを参照。

### Pro（月額300円 / 年額3,000円 / 買い切り5,800円）
全制限タイプ + グループ管理が使えるローカル完結プラン。

- **全7制限タイプ解放**
- 単体ルール: **無制限**
- **グループ作成**（カスタム + プリセットグループ）
- **グループ単位のON/OFF**
- ストリーク機能（ヒートマップ + 数値）
- 一時解除（バイパス）
- ロックモード（パスワード保護）
- 優先サポート

**目的**: 柔軟な制限設定を求めるパワーユーザー向け。

**買い切りについて**:
- 買い切り後に追加されるPro機能は利用可能
- Cloud機能（クラウド同期）は含まない

### Cloud（月額480円 / 年額4,800円）
Pro全機能 + クラウド同期。**サブスクのみ（買い切りなし）**。

- Pro全機能に加えて:
- **デバイス間リアルタイム同期**（ブロックリスト・ストリーク・DailyStats・表示設定）
- **クラウドバックアップ**（データ消失防止）
- Googleアカウントでログイン

**目的**: 複数PCを使うユーザー向け。Firebaseの運用コストをカバー。

**買い切りを提供しない理由**: Firebaseの運用コストが継続的にかかるため、サブスクで回収する。

**年額の割引**: 年額にすると2ヶ月分お得（全プラン共通で17%OFF）

## 認証とプランの関係

| プラン | 認証 | データ保存先 |
|--------|------|-------------|
| Free | 不要 | chrome.storage.local のみ |
| Pro | Googleログイン必須（ライセンス管理のため） | chrome.storage.local のみ |
| Cloud | Googleログイン必須 | chrome.storage.local + Firestore |

### オフライン時のライセンス検証
- ライセンス情報をローカルにキャッシュ
- 最終検証から**30日間**はオフラインでもPro/Cloud機能を有効に維持
- 30日超過後はFree機能に制限（データは保持、再ログインで復帰）

## 決済

### Stripe（メイン）
- 拡張の「Proにアップグレード」ボタン → **Stripe Checkoutページ**をブラウザで開く
- 決済完了 → Stripe Webhook → Firestoreにライセンス反映
- 拡張がFirestore経由でプラン変更を検知
- サブスク管理（プラン変更・解約）もStripeのカスタマーポータルで完結

### 決済フロー

```
拡張「Proにアップグレード」
  ↓
ブラウザでStripe Checkoutページを開く
  ↓
ユーザーがカード情報を入力・決済
  ↓
Stripe Webhook → Cloud Functions → Firestoreにライセンス書き込み
  ↓
拡張がFirestoreの変更を検知 → Pro/Cloud機能を有効化
```

### ルール
- **Pro限定タイプ**（回数・時間・クールダウン・遅延・位置情報）→ 自動ロック（動作停止、鍵アイコン表示）

## ライセンス管理

### ライセンス判定方式
- Firestoreのライセンスに `plan: "free" | "pro" | "cloud"` を保存
- 拡張側は `plan` の値だけで機能解放を判定（フィーチャーフラグ不要）
- P2で追加される位置情報・ストリークも、`plan: "pro"` なら自動的に使える
- 拡張のアップデートだけで新機能が解放される

### オフラインキャッシュ
- ライセンス情報は `chrome.storage.local` にキャッシュ
- キャッシュに `lastVerified` タイムスタンプを保存
- 最終検証から30日間はオフラインでもPro/Cloud機能を有効に維持
- 30日超過後はFree機能に制限（データは保持、再ログインで復帰）
- ブラウザデータ消去時はキャッシュ消失 → 再ログインが必要

## 価格フェーズ（段階的値上げ）

有料ユーザー数に応じて4段階で値上げする。既存ユーザーは旧価格を維持。

### 価格テーブル

| フェーズ | 条件 | Pro月額 | Pro年額 | Pro買い切り | Cloud月額 | Cloud年額 |
|---------|------|--------|--------|-----------|----------|----------|
| **A** | ローンチ〜500名 | 300円 | 3,000円 | 5,800円 | 480円 | 4,800円 |
| **B** | 500〜1,000名 | 380円 | 3,800円 | 7,800円 | 580円 | 5,800円 |
| **C** | 1,000〜5,000名 | 480円 | 4,800円 | 9,800円 | 780円 | 7,800円 |
| **D** | 5,000名〜 | 980円 | 9,800円 | 14,800円 | 1,480円 | 14,800円 |

※ 年額: 月額の10ヶ月分（2ヶ月無料、17%OFF）
※ 買い切り: A〜Cは年額の2倍、Dのみ年額の1.5倍

### Phase Dのトライアル特別価格

Phase Dに到達後、新規ユーザーの7日間トライアル期間中に契約すると特別価格で購入可能。
980円/1,480円は実売価格の実績があるため、割引表示は法的に問題なし。

| | 通常価格（D） | トライアル特別価格 |
|---|---|---|
| Pro月額 | 980円 | **480円** |
| Pro年額 | 9,800円 | **4,800円** |
| Pro買い切り | 14,800円 | **9,800円** |
| Cloud月額 | 1,480円 | **780円** |
| Cloud年額 | 14,800円 | **7,800円** |

**Phase A〜Cではトライアル特別価格なし**（価格自体が十分安いため、「現在の価格は今だけです。今後値上げ予定」で訴求）

### 価格設計の原則
- 価格はFirestoreで管理し、**拡張のコードにハードコードしない**
- Stripeの価格ID（price_xxx）をFirestoreに保存し、拡張が参照する
- 価格変更時はFirestore + Stripeを更新するだけで即反映

### 仕組み
- Firestoreに `paidUserCount`（累計有料ユーザー数）を管理
- 閾値到達時にFirestoreの現在フェーズを更新 → 新規ユーザーに新価格を適用
- 既存サブスクは旧価格のまま自動継続（Stripeのgrandfathering）
- 買い切りユーザーは値上げの影響を受けない
- Stripeでフェーズごとにprice_idを作成し、登録時点のフェーズで課金
- Phase D到達後、トライアル期間中のユーザーにはStripeクーポンで特別価格を適用

## トライアル管理

段階的に管理レベルを強化するハイブリッド方式。

### Phase A〜C（MVP〜P1初期）: ローカル管理

トライアルの目的は「Pro機能を体験してもらう」。濫用されても損失は小さいため、シンプルに実装。

**仕組み:**
- `chrome.storage.local` に `trialStartDate` を保存
- `chrome.runtime.onInstalled` でインストール時に記録
- 現在日 < trialStartDate + 7日 → Pro機能有効
- 期限切れ → Freeに移行 + ダウングレードUI表示

**濫用リスク:**
- 拡張を再インストールするとトライアルリセット可能
- ただし設定も全て消えるため手間がかかる
- 濫用する層はそもそも課金しないため、対策コストに見合わない

### P1以降（Google認証導入後）: Firestore管理

Google認証が入ったタイミングで、新規ユーザーのトライアルをFirestoreで管理に移行。

**仕組み:**
- 新規ユーザーがGoogle ログインした場合:
  - Firestoreに `trialStartDate` を保存（Googleアカウント単位）
  - 再インストールしてもトライアルリセット不可
- 未ログインユーザー:
  - 従来通りローカル管理（ログインしない限り濫用可能）
  - ただしPro機能を使い続けたいなら結局ログイン+課金が必要

### Phase D（トライアル割引導入後）: ログイン必須

トライアル割引（980円→480円等）は金額が大きいため、濫用防止を強化。

**仕組み:**
- トライアル自体はローカルでも開始可能（Pro機能の体験）
- **トライアル割引価格での購入にはGoogleログイン必須**
- Firestoreでトライアル割引の利用済みフラグを管理
- 1つのGoogleアカウントにつきトライアル割引は1回のみ

```
新規インストール
  ↓
7日間Proトライアル開始（ログイン不要）
  ↓
トライアル中に「特別価格で購入」をクリック
  ↓
Googleログインを要求
  ↓
Firestoreでトライアル割引未使用を確認
  ↓
Stripe Checkoutで割引価格を適用
  ↓
Firestoreにトライアル割引使用済みフラグを記録
```

### まとめ

| 段階 | トライアル開始 | トライアル管理 | 割引購入 |
|------|-------------|-------------|---------|
| MVP〜P1初期 | ログイン不要 | ローカル | - |
| P1以降 | ログイン不要 | ログイン済→Firestore / 未→ローカル | ログイン必須 |
| Phase D | ログイン不要 | ログイン済→Firestore / 未→ローカル | ログイン必須（割引は1回限り） |
