# Phase 1: 完全ブロック + UI

最小限の「使える状態」を作る。完全ブロック1タイプで全画面を通しで動かす。

---

## ゴール

- 設定ページからサイトを追加すると、そのサイトがブロックされる
- ポップアップでブロック状況を確認・ON/OFFできる
- ブロック時にブロックページが表示される
- プリセットから一括追加できる

---

## タスク

### 1-1. 完全ブロック機能
- `src/background/rule-sync.ts`: 既存の `rules.ts` をリファクタ
  - `RestrictionConfig` の `type: 'full_block'` のみ対応（他タイプは後続Phaseで拡張）
  - `syncRules()` を新しい型に合わせて更新
- `src/background/index.ts`: rule-syncを使うように更新
- ブロックページ改善:
  - 制限タイプ別メッセージの土台を作る（Phase 1では完全ブロックのメッセージのみ）
  - 設定ページへのリンク

### 1-2. 設定ページ（Options）
- ブロックリスト一覧表示（ドメイン・制限タイプ・ON/OFFトグル・削除ボタン）
- サイト追加ダイアログ:
  - ドメイン入力
  - 制限タイプ選択UI（Phase 1では完全ブロックのみ選択可、他は「Coming Soon」表示）
- プリセットから一括追加（カテゴリ選択 → 完全ブロックで追加、重複スキップ）
- バリデーション（空文字、重複ドメインのチェック）

### 1-3. ポップアップ
- ブロック状況サマリー（「ブロック中: N件」）
- ブロックルール一覧 + ON/OFFトグル
- サイト追加（ドメイン入力 → 完全ブロックで追加）
- プリセットから追加
- 設定ページへのリンク

## 完了条件

- 設定ページでサイトを追加 → ブロックが即座に有効になる
- ポップアップでON/OFF切り替え → ブロックが即座に反映される
- ブロック対象サイトにアクセス → ブロックページが表示される
- プリセットから一括追加 → 重複なく追加される
- `npm run build` が通る
- `npm test` が通る
