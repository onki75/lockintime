# ストリーク

制限を守った連続日数を記録・表示する。モチベーション維持のための機能。
サイト別 + 全体の両方を追跡し、ブロックページに表示する。

---

## 機能

- **サイト別ストリーク**: そのサイトの制限を守った連続日数
- **全体ストリーク**: 全ルールの制限を守った連続日数
- **表示形式**: GitHub草風ヒートマップ or 数値（設定で切替）
- **表示場所**: ブロックページ
- **最長記録**: 全期間での最長連続日数を記録

---

## 成功の定義

「成功 = 制限内に収まった」。制限タイプ別:

| タイプ | 成功 | 失敗 |
|--------|------|------|
| 完全ブロック | アクセスしなかった | ブロックページが表示された |
| 使用時刻制限 | 制限時間帯にアクセスしなかった | 制限時間帯にブロックされた |
| 使用回数制限 | 上限内でアクセス | 上限を超えてブロックされた |
| 使用時間制限 | 上限内で閲覧 | 上限を超えてブロックされた |
| クールダウン | クールダウン中にアクセスしなかった | クールダウン中にブロックされた |
| 遅延アクセス | 常に成功（ブロックではないため） | - |
| 位置情報制限 | 制限場所でアクセスしなかった | 制限場所でブロックされた |

※ 遅延アクセスは「待機後にサイトを閲覧」しても成功扱い。ストリーク対象だが失敗にはならない。

### Cloud同期時のマージルール
- 同じ日にデバイスAで成功、デバイスBで失敗 → **失敗が優先（AND）**
- どちらかのデバイスで制限を破ったら、その日は失敗

## タスク

### ロジック
- `src/lib/streak.ts` 新規作成
  - ストリーク判定:
    - サイト別: その日、対象サイトの制限を破らなかった → success
    - 全体: 全ルールでsuccessだった → global success
  - 連続日数の計算: StreakRecord[]を末尾から走査
  - 最長記録の計算: 全期間での最長連続success

### 日次更新
- DailyStatsの日次リセット時にストリーク判定を実行
- 前日のDailyStatsを元に StreakRecord を生成
- StreakDataに追記

### ヒートマップコンポーネント
- `src/components/Heatmap.tsx` 新規作成
  - GitHub草風のグリッド表示（12週分 = 84日）
  - 日ごとに success → 緑、failure → 灰色
  - 連続日数が長いほど緑が濃くなる（4段階）
  - **アクセシビリティ**: 色だけでなくパターン（斜線 = 失敗）+ ツールチップ（「3月16日: 成功」）で情報を伝える（WCAG 2.1 AA準拠）

### UI
- ブロックページにストリーク表示:
  - サイト別（連続日数 + ヒートマップ or 数値）
  - 全体（連続日数 + ヒートマップ or 数値）
  - 表示形式の切替ボタン
- 設定ページ:
  - 「表示設定」で streakDisplayMode 切替
