# 完全ブロック

常時アクセス不可にする最も基本的な制限タイプ。
MVPのコア機能であり、UI一式（ポップアップ・設定ページ・ブロックページ・プリセット）もここで作る。

---

## 機能

- ブロックリストにドメインを追加すると、そのサイトへのアクセスがブロックページにリダイレクトされる
- ルールごとにON/OFF切替可能
- プリセット（SNS/動画/ニュース/漫画/掲示板）から一括追加

---

## Chrome API

- `declarativeNetRequest` — 動的ルールでサイトをブロック（リダイレクト）

---

## タスク

### 基盤整理
- `src/lib/types.ts` に全型定義を集約
- `src/lib/storage.ts` のCRUD関数を新型に対応
  - `getSettings()` / `saveSettings()`
  - `addRule(url, restriction)` / `removeRule(id)` / `toggleRule(id)` / `updateRule(id, partial)`
- 既存の `focusMode` / `focusDuration` を削除

### ブロック機能
- `src/background/rule-sync.ts`: 既存の `rules.ts` をリファクタ
  - `full_block` タイプのルール同期
  - `syncRules()` を新しい型に合わせて更新
- `src/background/index.ts`: rule-syncを使うように更新

### 設定ページ（Options）
- ブロックリスト一覧（ドメイン・制限タイプ・ON/OFFトグル・削除ボタン）
- サイト追加ダイアログ（ドメイン入力 + 制限タイプ選択、MVP段階では完全ブロックのみ）
- プリセットから一括追加（カテゴリ選択 → 完全ブロックで追加、重複スキップ）
- バリデーション（空文字、重複ドメイン）

### ポップアップ
- ブロック状況サマリー（「ブロック中: N件」）
- ブロックルール一覧 + ON/OFFトグル
- サイト追加（ドメイン入力）
- プリセットから追加
- 設定ページへのリンク

### ブロックページ
- ブロック理由の表示（制限タイプ別メッセージの土台）
- 設定ページへのリンク

## オンボーディング（初回インストール時）

初回インストール時にセットアップウィザードを表示。空のポップアップではなく、すぐに価値を体験してもらう。

### フロー

```
Step 1: ようこそ画面
┌────────────────────────────┐
│ LockInTime へようこそ！      │
│                            │
│ 集中を妨げるサイトを         │
│ ブロックしましょう。         │
│                            │
│        [はじめる]           │
└────────────────────────────┘

Step 2: プリセット選択
┌────────────────────────────┐
│ ブロックしたいカテゴリを     │
│ 選んでください（5件まで）    │
│                            │
│ ☑ SNS (6)                  │
│ ☐ 動画 (5)                 │
│ ☐ ニュース (4)              │
│ ☐ 漫画・小説 (6)            │
│ ☐ 掲示板・まとめ (4)        │
│                            │
│    [スキップ]  [次へ]       │
└────────────────────────────┘
※ Free上限5件のプリセット選択UIと連携

Step 3: 完了
┌────────────────────────────┐
│ 設定完了！                   │
│ 3件のサイトをブロック中       │
│                            │
│    [ポップアップを開く]      │
└────────────────────────────┘
```

- `chrome.runtime.onInstalled` でインストールを検知 → オンボーディングページを開く
- スキップ可能（いつでも設定ページから操作可能）
- 完了後はフラグを保存し、2回目以降は表示しない

## プライバシーポリシー・利用規約

Chrome Web Store公開に必須。MVPリリース時に用意する。

- プライバシーポリシー: 収集するデータ（なし or 最小限）、保存場所、第三者共有の有無
- 利用規約: 免責事項、サービス範囲
- ホスティング: GitHub Pages等でWebページとして公開
- manifest.jsonのdeveloper URLに設定

---

## 完了条件

- 設定ページでサイトを追加 → ブロックが即座に有効になる
- ポップアップでON/OFF切り替え → ブロックが即座に反映される
- ブロック対象サイトにアクセス → ブロックページが表示される
- プリセットから一括追加 → 重複なく追加される
- 初回インストール時にオンボーディングが表示される
- プライバシーポリシー・利用規約のページが公開されている
- `npm run build` / `npm test` が通る
