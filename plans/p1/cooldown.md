# クールダウン

アクセス後に一定時間空けないと再アクセスできない制限。
「X見たら次は30分後まで不可」のようなユースケースに対応。

---

## 機能

- サイトにアクセスした時刻を記録
- 最終アクセスから設定時間が経過するまで再アクセスをブロック
- クールダウン経過後は再びアクセス可能（アクセスするとまたクールダウン開始）

---

## Chrome API

- `chrome.tabs` — アクセス検知（tab-tracker共通基盤）
- `chrome.storage` — DailyStats.lastAccess保存

---

## タスク

### Background
- `tab-tracker.ts` の共通基盤を利用（使用時間制限と共有）
- `lastAccess` はDailyStatsとは**別管理**（日次リセットの影響を受けない）
- `lastAccess` は `chrome.storage.local` に独立して保存
- アクセス時に `lastAccess + cooldownMinutes > 現在時刻` → ブロック
- **クールダウンは日をまたいでも継続**（23:50にアクセス、60分クールダウン → 0:50まで有効）
- `rule-sync.ts` を拡張: `cooldown` タイプ対応

### UI
- サイト追加ダイアログ: クールダウン分数の入力
- ポップアップ: 「あと20分後」のようなステータス表示
- 設定ページ: 編集ダイアログでクールダウン分数の変更
- ブロックページ: 「あと15分後にアクセスできます」メッセージ

## 完了条件

- アクセス後、設定時間内の再アクセスがブロックされる
- クールダウン経過後は再びアクセスできる
- 再アクセス時に新たなクールダウンが開始される
- `npm run build` / `npm test` が通る
