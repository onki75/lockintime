# ロックモード（パスワード保護）

設定変更やブロック解除にパスワードを要求する。
ユーザーが衝動的にブロックを解除してしまうのを防ぐ。

---

## 機能

- パスワードを設定すると、以下の操作にパスワード入力が必要になる:
  - ルールの無効化・削除
  - 一時解除の利用
  - ロックモード自体の解除
- パスワード未設定時は従来通り自由に操作可能
- 拡張の無効化/削除はブラウザ側の操作なので防げないが、「拡張が無効化されました」の通知は可能

---

## 制限事項

Chrome拡張の技術的制約として、以下は防げない:
- ブラウザの拡張管理画面（chrome://extensions）からの無効化・削除
- シークレットモードでのアクセス
- 別ブラウザの使用

これらは完全なペアレンタルコントロールではなく、**自制支援**の位置づけ。

---

## タスク

### パスワード管理
- パスワードはハッシュ化して `chrome.storage.local` に保存
- SHA-256等でハッシュ化（平文保存しない）

### UI
- 設定ページに「ロックモード」セクションを追加
  - パスワード設定/変更/解除
- パスワード入力ダイアログ（保護された操作の前に表示）

### パスワード回復（時間遅延アンロック）

パスワードを忘れた場合、24時間待つことでロックを解除できる。
衝動的な解除を防ぎつつ、永久にロックアウトされることを防ぐ。

```
┌──────────────────────────────────────┐
│ ロック解除リクエスト                   │
├──────────────────────────────────────┤
│ パスワードなしでロックを解除するには   │
│ 24時間待つ必要があります。            │
│                                      │
│ リクエスト後、24時間経過すると        │
│ ロックが自動解除されます。            │
│ （いつでもキャンセル可能）            │
│                                      │
│       [キャンセル] [リクエスト]       │
└──────────────────────────────────────┘
```

```
┌──────────────────────────────────────┐
│ ロック解除待機中                      │
├──────────────────────────────────────┤
│ ロック解除まであと: 18時間32分        │
│                                      │
│ [キャンセル]                          │
└──────────────────────────────────────┘
```

**仕組み:**
- 「パスワードを忘れた」→ 24時間カウントダウン開始
- `chrome.storage.local` に `unlockRequestedAt` を保存
- `chrome.alarms` で24時間後にロック解除
- カウントダウン中はいつでもキャンセル可能（パスワード入力でも即解除可能）
- 解除後、パスワードはリセットされる（再設定が必要）

---

## 完了条件

- パスワードを設定するとルール変更にパスワードが必要になる
- パスワードなしでは保護された操作ができない
- パスワードがハッシュ化されて保存される
- パスワード忘れ時に24時間遅延アンロックで回復できる
- 遅延アンロックをキャンセルできる
- `npm run build` / `npm test` が通る
